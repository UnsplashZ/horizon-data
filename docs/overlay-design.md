# horizon-data 覆盖层设计与路线图

> 整理日期：2026-06-16

## 1. 产品目标

Windows 上玩 FH6 时的**透明置顶屏幕覆盖仪表**。读 FH6 的 UDP Data Out（见
`forza-horizon-6-telemetry.md`），把关键信息显示在屏幕上自定义位置，替代右下角看着费劲的原生表。

## 2. 功能清单

### 核心（默认开启）
- **时速**（`Speed`×3.6 km/h）
- **转速**（`CurrentEngineRpm` + `EngineMaxRpm`，含换挡灯）
- **档位**（`Gear`，0=R/11=N）

### 可选模块（可逐个开关）
- **输入可视化**：油门 `Accel`(315)/刹车 `Brake`(316)/转向 `Steer`(320) 踏板条 + 方向条
- **摩擦力/抓地可视化**：四轮 `TireCombinedSlip`(180–192)，≈1 为抓地极限，超限变色/告警
- **G力可视化**：`AccelerationX/Z`(20/28) ÷9.81 的 G-G 圆点图

### 后续可扩展（同一包已含数据，白送）
- 圈速/最佳圈/分段、Boost、马力机(Power/Torque)、四轮胎温、悬挂行程、赛道小地图(Position)

## 3. 关键交互/视觉决策

- **透明置顶窗口**，无边框，点击穿透（不挡游戏操作）。
- **透明度可调**：用滑块控制。默认值取「既不挡视觉、又清晰」——
  HUD 背景半透明深色（约 alpha 0.35–0.5），文字/指针保持高不透明（≈1.0），
  全局再叠一个默认 ~85–90% 的整体透明度滑块。
  → 优先用 **CSS/背景 alpha** 控制（文字始终清晰），而非整窗 alpha（会把文字也淡化）。
- **位置自定义**：各模块独立、可拖动；位置/开关/透明度持久化到本地配置文件。
- **编辑/锁定双模式**：编辑模式可拖动+穿透关闭；锁定模式点击穿透，纯显示。

## 4. 打包要求（Windows）

- 产出 **安装版 .exe**（安装到 Program Files）**和绿色便携版**（解压即用，配置放程序同目录）。
- 体积/占用要小（与 GPU 重负载游戏同时运行）。
- 注意：游戏**全屏独占**下普通置顶窗口可能被盖住 → 建议游戏用**无边框窗口化**；
  或把仪表放到第二屏/平板则无此限制。

## 5. 技术选型（建议见下）

- 首选 **Tauri**（Rust + Web 前端，WebView2）：产物小（便携 exe 几 MB～十几 MB）、
  占用低、原生支持透明窗口/置顶/点击穿透(`set_ignore_cursor_events`)，
  可同时产出 NSIS 安装包与便携 exe。
- 备选 **Electron**：生态成熟、electron-builder 一键出安装包+便携版，但体积大（~150MB）、占用高。
- 备选 **C#/WPF/WinUI**：原生覆盖体验最好、单 exe 自包含，但仅 Windows 开发、Web 化仪表不便。

### 构建注意
- 开发机是 macOS，但目标是 Windows。**Tauri 跨平台编译到 Windows 较麻烦**，
  推荐用 **GitHub Actions 的 windows runner** 出正式 .exe（或在一台 Windows 机器上构建）。
- Electron-builder 可在 mac 上直接产 Windows 包，但仍建议 CI 出正式版。

## 6. 路线图

1. **UDP 接收 + 解析 demo**（验证 FH6 正常发包，打印 rpm/速度/档位）
2. 选定栈，搭透明置顶 + 点击穿透的空覆盖窗口
3. 核心三件套（速度/转速/档位）+ 透明度滑块 + 拖动定位 + 配置持久化
4. 可选模块（输入/摩擦/G力）
5. 打包：NSIS 安装版 + 便携版，CI 出 release
